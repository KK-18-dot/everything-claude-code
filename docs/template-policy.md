# Template Policy

This environment uses a **global base + thin project overlays** model.

- Global base: `~/.claude/`
- Orchestra overlay source: `~/.claude/templates/orchestra/`
- Project selection: `CLAUDE.local.md` + `.claude/project-mode`

## Template Locations

| Template | Source | Contents |
|----------|--------|----------|
| global base | `~/.claude/` | personal rules, commands, skills, agents, docs |
| orchestra overlay | `~/.claude/templates/orchestra` | `.codex`, `.gemini`, thin `.claude/` overlay, project `.mcp.json` |
| ecc | marker only | no project template copy |
| hybrid | global ecc base + orchestra overlay | no separate full template |

## Update Workflow

### Manual Update (Recommended)

```bash
claude-update-templates
# or
~/.claude/scripts/update-templates.sh
```

This refreshes locally managed overlay assets.

### Automatic Updates

Not implemented. Templates are updated only when you explicitly request.

**Rationale**: Auto-sync could break working projects or introduce untested changes.

## Applying Updates to Existing Projects

After updating templates, you can re-apply to a project:

```bash
cd ~/Projects/my-project
claude-route . --force
```

This will:
- Re-sync orchestra overlay assets if in orchestra/hybrid mode
- Refresh generated overlay entries in `.mcp.json`
- NOT copy a project `CLAUDE.md`
- NOT replace unrelated project MCP servers

To fully reset a project overlay:
```bash
rm -rf .codex .gemini CLAUDE.local.md
rm -f .claude/project-mode .mcp.json
claude-route . --mode <mode>
```

## hooks.json Duplication Warning

### The Problem

Claude Code v2.1+ automatically loads `hooks/hooks.json` from installed plugins. If you:
1. Install ecc as a plugin (`/plugin install`)
2. AND have local hooks in `~/.claude/hooks/hooks.json`

You may get:
```
Duplicate hooks file detected: ./hooks/hooks.json resolves to already-loaded file
```

### Solutions

**Option A: Use plugin mode only (no local hooks)**
```bash
# Remove local hooks
rm ~/.claude/hooks/hooks.json
# Use plugin
claude /plugin install everything-claude-code
```

**Option B: Use local mode only (no plugin)**
```bash
# Keep local hooks
# Do NOT install ecc as plugin
# Use template copy approach (this repo's default)
```

**Option C: Merge carefully**
```bash
# Disable plugin hooks
# In ~/.claude/settings.json:
{
  "disabledPluginHooks": ["everything-claude-code"]
}
```

### This Repo's Approach

This repo uses **Option B** (local mode) by default:
- Global behavior lives in `~/.claude/`
- Thin orchestra assets live in `~/.claude/templates/orchestra/`
- Assets are copied or merged to projects only when the mode requires them
- No plugin installation required
- Full control over hooks

## CLAUDE.local.md Policy

### Purpose
`CLAUDE.local.md` stores the project mode marker and is:
- Auto-generated by `claude-route`
- Read by Claude Code (via global CLAUDE.md directive)
- Git-ignored (personal config, not shared)

### Format
```markdown
<!-- claude_env: ecc -->
Mode: ecc (ecc|orchestra|hybrid). Follow this mode for commands, skills, and integrations.

(optional user notes below)
```

### Personal-Only Assumption

This environment assumes solo use. `CLAUDE.local.md` and `.claude/project-mode`
exist to steer your own local workflow, not to coordinate with teammates.
