# Verification First

## Rule

Before implementation, define verification criteria.
After implementation, run verification and report the exact result.

## Required Output

For each non-trivial task, include:

1. **Verification method**: test command, build command, lint command, or UI check steps
2. **Expected result**: what counts as success
3. **Observed result**: pass/fail and key output

## Minimum Standard

- New behavior: add or run focused tests
- Bug fix: include reproduction and non-regression check
- UI change: include screenshot or explicit visual checklist
- If verification cannot run locally, state why and provide manual fallback steps
