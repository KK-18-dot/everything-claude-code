# Default Idea Ranking Rubric

Score each dimension from `1` to `5`.

## Dimensions

- `Leverage`: How much repeated value this creates once built
- `Frequency`: How often the problem appears in real usage
- `Build Cost`: Lower effort scores higher
- `Maintenance Cost`: Lower ongoing burden scores higher
- `Strategic Fit`: How well it matches the user's actual priorities
- `Risk`: Fewer blockers, dependencies, or fragility scores higher

## Default Weights

- `Leverage`: 25%
- `Frequency`: 20%
- `Build Cost`: 15%
- `Maintenance Cost`: 15%
- `Strategic Fit`: 15%
- `Risk`: 10%

## Interpretation

- `4.0+`: build soon
- `3.0-3.9`: worthwhile, but not urgent
- `2.0-2.9`: only if it unlocks something else
- `<2.0`: skip unless constraints change
