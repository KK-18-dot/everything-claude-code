# README Onboarding Checklist

Use this checklist when auditing or updating a repository onboarding surface.

## Required Facts

- Project name
- One-sentence purpose
- Primary stack
- How to run locally
- How to build or package
- How to test or verify

## Useful Additions

- Repo status or current phase
- Important directories
- Environment setup at a high level
- Links to deeper docs

## Common Problems

- Commands in README no longer match `package.json` or build files
- Setup requires hidden prerequisites
- README describes features that are no longer present
- Status is stale and misleads continuation work

## Good README Shape

1. Overview
2. Stack
3. Commands
4. Environment Notes
5. Current Status or Roadmap
6. References
