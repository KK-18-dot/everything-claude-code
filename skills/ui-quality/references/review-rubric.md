# UI Review Rubric

## Required pre-build notes

### Visual thesis

One sentence describing mood, material, and energy.

### Content plan

What each section is trying to do.

### Interaction thesis

2-3 motions or interaction ideas that actually change the feel of the page.

## Review Axes

### Design quality

Does the UI feel coherent as a whole rather than assembled from parts?

### Originality

Are there deliberate visual decisions, or does it look like template/default output?

### Craft

Are spacing, typography, contrast, alignment, and state handling technically solid?

### Functionality

Can a user understand the screen and complete the main action without guessing?

## Anti-Slop Checks

- no generic stat-card soup by default
- no hardcoded random accent colors
- no decorative gradients unless justified by the product language
- no fixed/floating layer overlap with primary content
- no unexplained visual flourish that weakens hierarchy
