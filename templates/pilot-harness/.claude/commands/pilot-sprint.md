# Pilot Sprint

Use the project-local pilot lane instead of improvising in chat.

## Workflow

1. Read `.claude/pilot/spec.md`
2. Read the active sprint contract
3. Execute only the bounded sprint scope
4. Update review and handoff artifacts
5. If the pilot itself is being judged, update `two-stage-decision.md`

## Rules

- keep the sprint narrow
- findings first
- artifact updates are required
- do not silently expand the scope
